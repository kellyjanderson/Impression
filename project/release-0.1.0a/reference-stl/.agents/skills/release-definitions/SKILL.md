# Release Definitions

Release definitions are the version-level planning layer above progression.

## Purpose

They answer:

* what the version is trying to accomplish
* which user-visible outcomes it is meant to surface
* which architecture and UI branches are in scope
* which specifications are planned to land in that version

## Relationship To Other Documents

* product definition explains product direction
* UI definitions explain visible behavior
* architecture explains system structure
* specifications define implementable units
* release definitions group a version holistically
* planning sequences final leaves for execution

## Recommended Structure

Release definitions should generally include:

* intent
* planned architecture
* planned UI definitions
* planned specifications
* version-level notes or exclusions

## Constraints

Release definitions should:

* stay version-oriented and holistic
* link to source documents instead of duplicating all their detail
* avoid becoming a second progression checklist
* stay anchored on surfaced user-facing outcomes for user-facing release goals
