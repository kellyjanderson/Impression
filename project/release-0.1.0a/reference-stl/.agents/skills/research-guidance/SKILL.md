# Research Guidance

Use this skill when research needs to become a durable local asset instead of a one-off answer.

## Use This For

* saving findings about tools, libraries, standards, and constraints
* preserving experiments and implementation notes
* distilling reusable reference material for later skills
* organizing research by topic with provenance

## Do Not Use This For

* architecture decisions themselves
* implementation specifications
* planning documents

## Purpose

Research documents preserve useful information that would otherwise fall out of working context.

They exist so that:

* researched information remains available after the current context is gone
* future work can build on prior research
* later local work can reuse durable reference material instead of rediscovering it

## Location

Store research in the nearest durable local research area for the work.

For project-oriented repositories, prefer:

```text
project/research/
```

Use subfolders when that helps organize topic areas.

## Workflow

1. Identify the topic and why it matters.
2. Store findings in a topic-oriented local structure.
3. Record implications, not only facts.
4. Include references or provenance.
5. Link the research to later work when it becomes active.

## Preferred Structure

Research documents should usually include:

* topic
* findings
* implications
* references
