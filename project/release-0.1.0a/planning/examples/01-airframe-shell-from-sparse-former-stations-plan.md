# 01 Airframe Shell From Sparse Former Stations

## Objective

Model a small airframe body that reads immediately as aircraft structure rather
than as a generic vase:

- blunt but aerodynamic nose
- slightly crowned cockpit shoulder
- full mid-body cabin section
- disciplined taper into an empennage-style tail

The example should prove that `0.1.0.a` can start from dense section evidence,
infer a cleaner internal explanation, retain topology-critical stations, and
report what it kept or refused.

## Release Delta

- `0.0.3a2`
  A strong surfaced loft can already be authored, but the modeler would rely on
  many manual sections and would have no durable reduced-progression story.
- `0.1.0.a`
  The same family can be represented with dense evidence plus fitted curve
  intent, hidden control stations, retained topology stations, and explicit
  reduction/refusal reporting.

## Visual Target

The preferred visual read is a compact light-aircraft fuselage shell, halfway
between a glider nose and a small composite UAV body.

Important reads:

- nose must not look spherical
- belly should feel flatter than the upper crown
- maximum width and maximum height should not occur at exactly the same station
- tail taper should feel disciplined and purposeful

## Model Decomposition

The model should be built from one outer shell profile family only.

No holes.
No satellite regions.
No split/merge topology.

This example is about section reduction and shape explanation, not topology
events.

## Profile Strategy

Each section should be generated by one helper:

`make_airframe_section(width, height, belly_flatness, crown_bias, nose_sharpness, tail_pinching, color)`

Implementation posture:

- create a 2D closed outline with `Path2D.from_points(...)`
- use an asymmetric point distribution
- keep the lower belly flatter than the roof
- use more points near the chine and nose quadrants than at the top center

The section should not be a pure ellipse.

It should be built from about `20` to `28` ordered points so it reads like a
designed fuselage former rather than a mathematically perfect oval.

## Spine And Progression Plan

Authored semantic spine:

- one `BSpline3D` or `Path3D` centerline
- mostly straight in plan
- slight upward crown through the cockpit shoulder
- slight downward return toward the tail cone

Suggested normalized progression values:

| Station ID | `t` | Longitudinal Role |
|---|---:|---|
| `nose_tip` | `0.00` | extreme nose termination |
| `nose_former` | `0.08` | establishes nose volume |
| `cockpit_break` | `0.20` | first strong crown / width gain |
| `shoulder_peak` | `0.34` | visual max height |
| `cabin_max` | `0.48` | visual max width |
| `aft_body` | `0.66` | start taper discipline |
| `tail_root` | `0.82` | strong narrowing, still stable |
| `tail_cone` | `0.94` | sharp taper read |
| `tail_tip` | `1.00` | tail termination |

## Important Station Plan

Suggested authored sparse station family:

| Station ID | Width | Height | Belly Flatness | Crown Bias | Notes |
|---|---:|---:|---:|---:|---|
| `nose_tip` | `4` | `5` | `0.05` | `0.30` | nearly pointed but still loftable |
| `nose_former` | `18` | `22` | `0.10` | `0.38` | establishes nose shoulder |
| `cockpit_break` | `34` | `38` | `0.18` | `0.46` | visible canopy lift |
| `shoulder_peak` | `44` | `51` | `0.22` | `0.58` | max height station |
| `cabin_max` | `54` | `47` | `0.24` | `0.50` | max width station |
| `aft_body` | `40` | `36` | `0.20` | `0.42` | taper begins without collapsing |
| `tail_root` | `24` | `20` | `0.12` | `0.34` | tail structure still readable |
| `tail_cone` | `11` | `10` | `0.08` | `0.22` | strong pinch |
| `tail_tip` | `3` | `3` | `0.00` | `0.10` | end condition |

Dense evidence plan for the reduction demonstration:

- derive `18` to `26` intermediate sections between the sparse stations
- perturb spacing so the dense stack looks like authored cleanup work rather
  than uniform resampling
- keep `shoulder_peak`, `cabin_max`, and `tail_root` as required topology
  stations

## Modeling Sequence

1. Author the sparse centerline as a `BSpline3D` or `Path3D`.
2. Build the sparse section family with `make_airframe_section(...)`.
3. Construct a surfaced `Loft(...)` from the sparse family for the hero result.
4. Build a second dense-evidence stack by sampling between sparse stations.
5. Wrap the same spine in `PathBackedProgression(...)`.
6. Attach the dense stations with `ProgressionStationAttachment.from_station(...)`.
7. Run dense descriptor preparation and station-derived fit candidate
   comparison.
8. Build `ReducedProgressionBundle.from_progression(...)`.
9. Assess structural preservation with explicit retained-station records.
10. Generate a shared diagnostic bundle and developer inspection artifact.

## Demo Composition

This example should eventually render as a three-part comparison:

- sparse authored hero shell
- dense evidence shell or wireframe station stack
- reduced progression / retained station diagnostic overlay

The sparse authored shell should be the hero image.
The dense stack and diagnostics should be support panels.

## Visual Review Loop For Execution

When we later build this model:

- render front, side, top, and isometric views after the first sparse loft
- send those renders to a visual-review sub-agent
- ask specifically whether the shape reads as aircraft structure rather than
  vase, fish, or rocket
- adjust the nose former, shoulder peak, and cabin max stations before adding
  the dense-evidence demonstration layer

The first visual gate is silhouette believability, not inference plumbing.

## Risks To Guard Against

- too much circularity in the cross-sections makes it read as a bottle
- too much roof bulge makes it read as a cartoon submarine
- too little belly flattening makes the reduction story visually unconvincing
- tail taper that begins too early makes the body look weak and toy-like
