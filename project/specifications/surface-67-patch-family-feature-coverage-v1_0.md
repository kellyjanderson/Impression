# Surface Spec 67: Patch-Family to Feature Coverage Matrix (v1.0)

## Overview

This specification defines how the chosen patch families map onto planned
modeling features.

## Backlink

Parent specification:

- [Surface Spec 22: V1 Patch Family Scope and Explicit Exclusions (v1.0)](surface-22-v1-patch-family-scope-v1_0.md)

## Scope

This specification covers:

- family-to-feature mapping
- required feature coverage by family
- known feature gaps created by family scope

## Behavior

This branch must define:

- which features each included family supports
- which features require multiple families
- which feature gaps remain because deferred families are excluded

## Constraints

- the matrix must be explicit
- unsupported feature coverage must not be implied
- the matrix must stay aligned with the program roadmap

## Refinement Status

Final.

## Child Specifications

None. Final leaf.

## Acceptance

This specification is complete when:

- family-to-feature mapping is explicit
- cross-family support obligations are explicit
- feature gaps caused by exclusions are explicit
